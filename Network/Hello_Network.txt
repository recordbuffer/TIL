> 네트워크 Network : 컴퓨터와 컴퓨터 사이를 연결한 것

네트워크 구조 (=토폴로지 Topology)
: 네트워크 요소들의 연결구조

- 망사형 : 모든 컴퓨터 장치 연결 
	→ 응답 시간 빠름	→ 통신회선 고장이 전체에 영향 없음 (=다른 회선으로 전송 가능)
	→ 공중 데이터 통신망에 이용 (대규모 네트워크에 유리)
	→ 구축 비용 큼

- 스타형 : 모든 컴퓨터 장치가 중앙 허브에 1:1로 연결
	→ 유지보수 확장 용이
	→ 중앙 허브의 안정성, 신뢰성 확보 필요

- 트리형 (계층형) : 계층적으로 연결 
	→ 분산 처리 시스템 구성 방식

- 링형 (루프형) : 인접한 컴퓨터 장치끼리 연결
	→ 하나 고장나면 통신 전체 불가
	
- 버스형 : 하나의 통신 회선에 여러대의 컴퓨터 장치 연결 
	→ 구조 간단, 설치 쉬움


네트워크 분류 
- 근거리 통신망 LAN (Local Area Network) : 지리적으로 제한된 건물 범위 / 주로 버스형, 링형, 스타형
- 도시 통신망 MAN ((Metropolitan Area Network) : 건물, 도시 범위 / 여러 LAN을 라우터로 상호 연결해 만듦
- 광역 통신망 WAN (Wide Area Network : 지역-지역, 국가-국가 연결

+
인터넷 서비스 제공자 ISP (Internet Service Provider) : KT, SKT, U+ 등
부가가치 통신망 VAN : 사업자로부터 통신 회선을 빌려 기존 정보에 가치를 더해 여러 이용자에게 판매하는 통신망
인터넷 프로토콜 음성 VoIP : 인터넷을 통한 은성 전화 통신

DMZ 네트워크 =비무장지대 : 외부에 공개하기 위한 네트워크 영역, 외부 네트워크와 내부 네트워크 사이 중간 지대 (서브넷)
- 웹서버 → 불특정 다수 외부 사용자에게 공개하는 서버
- 메일서버 → 외부 사용자와 메일 주고받기 위한 서버
- DNS 서버 → 외부에서 도메인 사용해 서버에 접속하기 위한 서버


클라우드 Cloud : 네트워크를 이용해 어디에서나 접근이 가능한 서버를 빌려주는 서비스 
온프레미스 On-premis : 클라우드 기술이 나오기 전 사내 전산실 서버를 직접 설치해 운영하는 서비스 방식


네트워크 데이터 교환방식
- 회선 교환 : 데이터 전송 전 물리적으로 통신 회선 연결 
	→ 고정된 대역폭으로 데이터 전송 	→ 전송 지연 거의 없음

- 축적 교환 
	- 패킷 교환 : 데이터를 일정할 길이의 패킷으로 잘라 전송	→ 최적 경로 선택하는 라우팅 기능
		→ 응답시간 빠름 (=대화형 데이터 통신에 적합)
		+ 패킷 Packet : 용량이 큰 데이터 패킷으로 분할해 전송 / 패킷 별로 순서 생성 > 도착 후 정렬

		⒜ 가상회선 방식 : 패킷의 경로 설정 후 순차적으로 전송
		⒝ 데이터그램 방식 : 특정 경로 설정 없이 전송 / 목적지에서 정렬


	- 메시지 교환 : 하나의 메시지 단위로 저장하고 전송 
		→ 메시지마다 전송 경로 달라 수신 주소 붙여서 전송함 	→ 전송 지연 시간 김


네트워크 규칙
- 프로토콜 Protocal : 통신을 위해 정해놓은 절차	  ex) HTTP,IP, TCP, UDP 등
	① 연결 설정 단계
	② 데이터 전송 단계
	③ 연결 해제 단계
- IP 주소 : 네트워크 통신을 위한 고유주소
- 포트 Port : 컴퓨터 내 프로세스로 가는 주소 (프로그램 식별 / 서버 구분)

→ 어떤 방식(프로토콜)로 어디에 있는 컴퓨터(IP주소)에 어떤 프로그램(port)을 연결할 것인가?


도메인 Domain : 사람들이 쉽게 네트워크를 식별할 수 있게 한것 (네트워크 영문 주소)
URL : 특정 웹 서버의 특정 파일에 접근하기 위한 경로 
	→  프로토콜://호스트(서버) 주소[;포트번호][/파일경로]
소켓 Socket : 네트워크 상 데이터를 입출력하기 위한 장치 / 프로세스 간 통신에 사용


> OSI 기본 참조 모델
7계층
⑦ 응용 계층 (=애플리케이션 계층)
⑥ 표현 계층 (=프레젠테이션 계층)
⑤ 세션 계층
④ 전송 계층 (=트랜스포트 계층)
-----------------------상위 계층 : 두 컴퓨터에서 실행되는 프로그램간 연결 설정 담당
-----------------------하위 계층 : 네트워크에서 두 컴퓨터 간의 연결 설정 및 데이터 전송 지원
③ 네트워크 계층
② 데이터 링크 계층
① 물리 계층


- 물리 계층 (1 계층 / L1) : 송수신하는 두 컴퓨터의 물리적 연결 및 전기 신호 제어 → 광 케이블 등
- 데이터 링크 계층 (2 계층 / L2) : 물리적 통신 경로(주소) 확립, 데이터 전송 오류 제어, 흐름 제어 → 이더넷, 스위칭 등
- 네트워크 계층 (3 계층 / L3) : 송신 데이터가 최적 경로 선택하도록 경로 설정, 주소 관리 → IP, 라우팅 등
- 전송 계층 (4 계층 / L4) : 신뢰할 수 있는 데이터 전송 보장, 송수신 전체 책임 → TCP, UDP 등
- 세션 계층 (5 계층 / L5) : 응용 프로그램 간 세션 통신 방식 결정 
- 표현 계층 (6 계층 / L6) : 송수신되는 데이터를 통신에 맞게 형식 변환 → 문자코드, 압축, 암호화 등
- 응용 계층 (7 계층 / L7) : 사용자와 애플리케이션 간 데이터 송수신 서비스 제공 → WWW, HTTP, Excel 등


각 계층 별 데이터 단위 (PDU)
- 데이터 링크 계층 (2 계층) → 프레임
- 네트워크 계층 (3 계층) → 패킷
- 전송 계층 (4 계층) → 세그먼트


+ TCP/IP 모델
7계층을 4계층으로 줄인 모델 
④ 응용 계층 (= ⑦ 응용 계층 + ⑥ 표현 계층 + ⑤ 세션 계층)
③ 전송 계층 (= ④ 전송 계층)
② 인터넷 계층 (= ③ 네트워크 계층)
① 네트워크 접속 계층 (= ① 물리 계층 + ② 데이터 링크 계층)


> 데이터 송수신

캡슐화 : 데이터 전송에 필요한 정보를 헤더에 붙여 보내는 것
역캡슐화 : 데이터 수신에 헤더를 제거하는 것


> 물리 계층 : 송수신하는 두 컴퓨터의 물리적 연결 및 전기 신호 제어
전기 신호 
- 아날로그 신호 : 물결 모양 신호, 연속적인 변화
- 디지털 신호 : 막대 모양 신호, 스위칭 작용

데이터 전송 매체 
- 유선 = 트위스트 페어 케이블, 광 케이블 등
- 무선 = 라디오파, 마이크로파, 적외선 등

트위스트 페어 케이블 =랜 케이블
- UTP 케이블 (Unshielded Twist Pair) : 구리선으로 꼬아 만든 실드로 보호되어 있지 않은 케이블 → 저렴함
- STP 케이블 (Shielded Twist Pair) : 금속호일이나 금속 매듭 같은 걸로 보호되어 있는 케이블 → 노이즈 제어 

+노이즈 : 케이블에 전기 신호 흐를 때 발생

허브 
① 전기 신호를 증폭하는 기능
② 노이즈로 변경된 신호를 되돌리는 기능
③ 컴퓨터 여러대를 서로 연결하는 장치 

→ 허브의 포트로 연결된 모든 컴퓨터에 데이터 전송함 (스스로 판단 못함)


> 데이터 링크 계층 : 물리적 통신 경로(주소) 확립, 데이터 전송 오류 제어, 흐름 제어
→ 네트워크 장비간 신호 주고받는 규칙 필요 

이더넷 : 랜에서 적용되는 규칙 / 허브에 연결된 컴퓨터와 데이터 주고받을때 사용하는 규칙
→ 여러 대의 컴퓨터가 데이터를 보낼 때 발생할 수 있는 충돌을 일어나지 않게 하는 구조
→ CSMA/CD
- CS : 데이터 송신 컴퓨터가 케이블에 신호 흐르는지 아닌지 확인
- MA : 케이블에 신호 없으면 데이터 송신
- CD : 충돌 발생하는지 확인

랜카드 : 네트워크 통해 데이터를 송수신하고 전기 신호로 변환하는 장치
→ MAC 주소 (Media Access Control Address) : 전 세계에서 유일한 번호, 물리 주소 할당됨 → 48비트 숫자
→ 데이터 링크 계층
	- 이더넷 헤더 : 목적지의 MAC 주소 + 출발지의 MAC 주소 + 유형 (프로토콜 종류 식별 번호)
	- 트레일러 =FCS : 데이터 전송 도중에 오류 발생하는지 확인

스위치 : 향상된 허브, 모든 컴퓨터에 데이터 전송하는 게 아니라 MAC 주소로 판단해 수신
→ MAC 주소 테이블에 포트와 함께 등록 
→ MAC 주소 등록되어 있는 컴퓨터에만 데이터 전송됨 (필터링)


> 네트워크 계층 : 송신 데이터가 최적 경로 선택하도록 경로 설정, 주소 관리
라우터 Router : 네트워크 간 데이터 전송하기 위한 장비 
→ 라우팅 : 목적지(IP주소)까지의 최적의 경로 결정하는 것
→ 라우팅 테이블에 경로 정보 등록 관리

→ 네트워크 계층 
	- IP 헤더 : 버전 + 헤더 길이 + 서비스 유형 + 전체 패킷 길이 + 일렬번호 + 조각상태 + 조각 위치 +  TTL + 프로토콜 + 헤더 체크섬 + 출발지 IP주소 + 목적지 IP주소

IP 버전 
- IPv4 : 8비트씩 4부분으로 . 으로 구분하는 총 32비트로 이루어진 주소 → 네트워크 부분 길이에 따라 A~E클래스로 구분
- IPv6 : 16비트씩 8부분으로 : 으로 구분하는 총 128비트로 이루어진 주소
	→  유니캐스트 / 멀티캐스트 / 애니캐스트

서브넷 : 분할된 네트워크


> 전송 계층 : 신뢰할 수 있는 데이터 전송 보장, 송수신 전체 책임
→ 데이터 전송 오류 점검하는 기능
→ 전송된 데이터의 목적지가 어떤 애플리케이션인지 식별하는 기능 

- 연결성 통신 : 신뢰할 수 있고 정확히 데이터를 전송하는 통신 방식 → TCP
- 비연결성 통신 : 신뢰성 보장 안하지만 효율성 높은 데이터 전송 통신 방식 → UDP

TCP (Transmission Control Protocol) 전송 제어 프로토콜
- TCP 헤더 : 출발지 포트 번호 + 목적지 포트 번호 + 일렬번호 + 확인 응답 번호 + 헤더 길이 + 예약 영역 + 코드 비트 + 윈도우 크기 + 체크섬 + 긴급 포인터 + 옵션 
→ 코드 비트 : (통신로) 연결의 제어 정보 기록됨 
	- SYN : 연결 요청
	- ACK : 확인 응답


데이터 송수신 전 연결 과정 = 3-way 핸드셰이크
① 연결 확립 요청 SYN →
② 연결 확립 응답 + 연결 확립 요청 ACK + SYN ←
③ 연결 확립 응답 SYN →

④ 연결 종료 요청 FIN →
⑤ 연결 종료 응답 ACK ←
⑥ 연결 종료 요청 FIN ←
⑦ 연결 종료 응답 ACK →

UDP (User Datagram Protocol) 사용자 데이터그램 프로토콜 
→ 동영상 스트리밍 등에서 사용 (전송 속도 빠름)
- 브로드캐스트 통신 : 랜에서 컴퓨터나 네트워크 장비에 데이터 일괄 전송함
- UPD 헤더 : 출발지 포트 번호 + 목적지 포트 번호 + 길이 + 체크섬

게이트 웨이 : 서로 다른 프로토콜끼리도 네트워크 통신 가능하도록 연결해주는 장치


> 응용 계층 : 사용자와 애플리케이션 간 데이터 송수신 서비스 제공
클라이언트 : 서비스를 요청하는 측
서버 : 서비스를 제공하는 측

대표적인 프로토콜 
- HTTP : 웹사이트 접속
- DNS : 도메인 해석 (www.naver.com > 192.xxx...)
- FTP : 파일 전송
- SMTP : 메일 송신
- POP3 : 메일 수신


> HTTP : 서버와 클라이언트가 인터넷상에서 데이터를 주고 받기  위한 무상태 프로토콜 방식 → 80번 포트 사용
+ 무상태 프로토콜 (Stateless Protocal) : 서버와 클라이언트가 요청-응답 후 연결이 끊겨 이전의 클라이언트의 정보를 모르게 되는 프로토콜 방식

→ 데이터 요청-응답 독립적인 관리 == 서버와 클라이언트의 최대 연결 수보다 많은 요청과 응답 처리할 수 있음 
→ 불특정 다수 대상 서비스 적합함


HTTP 작동방식
① 클라이언트 -- 연결 -- 서버

③ 클라이언트 <-- 응답 <-- 서버

HTTP 요청 메시지
- HTTP 요청 메서드
	- Get 방식 : 이미 존재하는 데이터 요청 / URI에 데이터 붙여 전송
	- POST 방식 : 새로운 데이터 생성 요청 / 요처 바디에 붙여 전송
	- PUT 방식 : 이미 존재하는 데이터 변경 요청
	- DELETE 방식 : 이미 존재하는 데이터 삭제 요청
- URL
- HTTP 버전


HTTP 응답 메시지 
- HTTP 버전
- HTTP 응답 코드
	- 200 : GET 요청 성공
	- 400 : Bad Request 일반 오류 코드
	- 404 : Not Found 요청 내용이 서버에 존재하지 않음
	- 500 : Internal Server Error 서버 오류
- 상태 메시지
- 날짜
- 웹 서버 이름
- 콘텐츠 타입, 길이


> DNS : URL을 IP주소로 변환하는 서비스 (도메인 이름 해석)

도메인 : IP 주소를 사람이 이해하기 쉬운 형태로 표현한 것 

DNS 통신 과정
① 컴퓨터에서 DNS 서버에 특정 도메인의 IP 주소 해석 요청
② DNS 서버에서 IP 주소 응답
③ 컴퓨터는 응답받은 IP 주소로 서버 접속


> FTP : 파일 전송 프로토콜
→ 파일 전송(업로드), 수신(다운로드), 삭제, 이름 변경 등의 작업 수행
→ 그림, 동영상 압축 파일 전송은 binary 모드에서 수행 / 텍스트 파일 전송은 ASCII 모드에서 수행


> 프록시 서버 Proxy Server : 방화벽을 가지고 있는 경우 외부와의 통신 위한 PC 사용자와 인터넷 사이 중계자 역할을 하는 서버
