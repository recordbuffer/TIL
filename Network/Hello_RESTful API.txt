RESTful API
REST : REpresentational State Transfer 
HTTP 통신의 장점을 최대로 활용할 수 있는 아키텍처로 소개

1) RESTful API 특징
① 클라이언트-서버 → 관심사 분리 = 클라이언트, 서버 역할 명확히 분리하는 것 
② 무상태성 → 클라이언트의 요청 정보 저장하지 않는 것
	→ 최대 연결 수보다 많은 요청, 응답 처리할 수 있음
③ 캐시 제어 : HTTP의 캐싱 기능 사용 가능 불가능 명시해야함
④ 유니폼 인터페이스 :  주어진 조건에 따라 URI 결정, 약속된 인터페이스 제약조건
	→ 리소스 식별
	→ 표현을 통한 리소스 조작
	→ 자기 서술 메시지	
	→ 애플리케이션 상태의 엔진으로서 하이퍼미디어
⑤ 계층형 시스템 : RSET 서버는 다중 계층으로  구성될 수 있음 → 관심사 분리
	→ 보안, 로드 밸런싱, 암호화, PROXY, 게이트 웨이 등 다양한 역할로 계층 구별 사용 가능
⑥ 주문형 코드 : 클라이언트가 전송한 자바스크립트 등 코드로 서버가 실행될 수 있어야 한다는 것
